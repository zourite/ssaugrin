Sauvegarder ses données dans Dropbox avec mysqldump

3 décembre 2015 Zourite

2 minutes

J’écris cet article suite à une mésaventure. Tous les disques durs de mon serveur sont tombés en panne, j’ai perdu près de deux ans de travail pour l’un de mes sites. Je suis paresseuse et faire des backups hebdomadaire via une interface ne m’enchante guère.

La solution qui permettra à cette mésaventure de ne plus se reproduire, c’est d’utiliser cron et mysqldump, ça ne me tente pas d’écrire un script pour automatiser l’envoie de mes données vers un serveur FTP. Le FTP is so hasbeen now, donc transférer tout ceci vers le cloud me parait être une bonne idée.

J’ai choisi Dropbox car le site fourni une application officielle pour Debian et son installation est très simple. Si vous n’avez pas de compte, je vous invite à vous inscrire : Se créer un compte Dropbox. Je pense ne pas être la seule à vouloir stocker mes backups dans Google Drive, plus loin dans l’article je vous dirais la solution que j’ai choisi pour transférer tout ceci vers Google Drive depuis Dropbox.

Les commandes ci-dessous vous permettent d’installer Dropbox et de sauvegarder vos données vers ce dernier. Il est important de démarrer Dropbox en arrière plan, si vous tuer le processus vos fichiers ne se synchroniseront plus.

Garder mes données uniquement dans Dropbox c’est prendre le risque de les perdres si il arrive un désastre de n’importe quel nature chez Dropbox. Donc pour bien consolider mon processus j’envoie mes données à Google Drive, grâce à IFTTT : Sync new files added to Dropbox to Google Drive
